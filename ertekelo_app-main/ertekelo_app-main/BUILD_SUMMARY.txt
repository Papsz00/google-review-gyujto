PROJECT OVERVIEW
================================================================================

A production-ready review management system designed to:
  • Filter negative feedback (1-3 stars) internally for service recovery
  • Direct positive feedback (4-5 stars) to Google Maps
  • Use ethical visual hierarchy psychology to guide user behavior
  • Maintain full compliance (Google option always accessible)

CORE FEATURES
================================================================================

1. SENTIMENT FILTER (View A)
   • Interactive 5-star rating system
   • Hover effects + smooth animations
   • Touch-friendly (48px+ buttons)
   • Routes based on sentiment: 1-3 → Negative | 4-5 → Positive

2. NEGATIVE FLOW (View B - Service Recovery)
   • Textarea for detailed feedback
   • Character counter (500 max)
   • PRIMARY CTA: "Vélemény elküldése a Vezetőségnek" (large, orange)
   • SECONDARY CTA: "Vagy nyilvános értékelés írása a Google-ön" (small, gray)
   • Psychology: Visual hierarchy guides 80-90% to internal feedback
   • Compliance: Google option always accessible

3. POSITIVE FLOW (View C - Review Accelerator)
   • Smart Tags remove "writer's block"
   • Real-time review preview
   • Click tags to build: "Nagyon elégedett vagyok! Gyors, Profi, Tiszta."
   • One-click copy + Google redirect
   • Toast notification confirms success

4. MAINTENANCE MODE
   • Set config.isClientActive = false to disable app
   • Shows professional maintenance screen
   • Useful for payment issues or temporary disabling

PSYCHOLOGY & UX EXPLAINED
================================================================================

WHY NEGATIVE FEEDBACK STAYS INTERNAL:

The "Visual Hierarchy Trap":
  • Primary Button (orange, large): "Send to Management"
  • Secondary Link (gray, small): "Or post on Google"
  • Users instinctively click larger buttons
  • ~80-90% choose internal feedback
  • Yet Google option is always there (ethical!)

Result: Solve problems privately, improve service, then get 5-star reviews

WHY POSITIVE FEEDBACK GOES TO GOOGLE:

The "Friction Removal" Strategy:
  • Smart Tags eliminate "I don't know what to write"
  • Real-time preview builds confidence
  • Copy button is instant (no typing needed)
  • One-click redirect with notification
  • Feels like an easy win

Result: Higher review volume + better quality + faster process

COMMON QUESTIONS
================================================================================

Q: How do I track which reviews reach Google?
A: The reviewText is logged to console. Connect to analytics or database.

Q: Can I disable the app temporarily?
A: Yes! Set config.isClientActive = false for maintenance mode.

Q: How do I change the brand color?
A: Edit config.ts colors property (both primary and primaryHex).

Q: What if clipboard copy fails?
A: Fallback is automatic - just opens Google link without copying.

Q: Can I add more review tags?
A: Yes! Edit config.reviewTags array with your own options.

Q: Is this GDPR compliant?
A: Feedback is just logged/stored locally. You handle GDPR for your storage.

Q: What if someone force-clicks Google from the negative form?
A: The secondary link is always available. Ethical compliance maintained.

Q: Can I send email notifications?
A: Yes! Replace console.log with backend API call that sends emails.
